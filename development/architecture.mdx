Khedma Market is built with modern web technologies focusing on type safety, developer experience, and performance.

## Tech stack

The application uses the T3 Stack as its foundation, combining:

<CodeGroup>
```json Frontend
"next": "^14.0.4"

"tailwindcss": "^3.4.1"
```

```json Backend
"@trpc/server": "^10.43.6"
"@prisma/client": "^5.10.1"
"next-auth": "^4.24.5"
```

```json Database
"prisma": "^5.10.1"
"mysql": (via DATABASE_URL)
```

### Core technologies

- **Next.js 14**: React framework with App Router for server-side rendering and routing
- **TypeScript**: Type-safe development across the entire stack
- **tRPC**: End-to-end typesafe APIs without code generation
- **Prisma**: Type-safe database ORM and migrations
- **NextAuth.js**: Complete authentication solution
- **Tailwind CSS**: Utility-first CSS framework
- **Radix UI**: Accessible, unstyled UI components

## tRPC API layer

Khedma Market uses tRPC for type-safe API communication between client and server.

### Router structure

The API is organized into domain-specific routers:

```typescript src/server/api/root.ts
export const appRouter = createTRPCRouter({
  user: userRouter,
  file: fileRouter,
  order: orderRouter,
  profile: profileRouter,
  category: CategoryRouter,
  gig: gigRouter,
  project: projectRouter,
  conversation: conversationRouter,
  job: jobRouter,
});

export type AppRouter = typeof appRouter;
```

### Context and procedures

Every tRPC procedure has access to context containing the database and session:

```typescript src/server/api/trpc.ts
export const createTRPCContext = async (opts: { headers: Headers }) => {
  const session = await getServerAuthSession();

  return {
    db,
    session,
    ...opts,
  };
};
```

<Accordion title="Public procedures">
  Available to all users, authenticated or not:

  ```typescript
  export const publicProcedure = t.procedure;
  ```

  Used for:
  - Fetching public gigs
  - Viewing freelancer profiles
  - Browsing categories
</Accordion>

<Accordion title="Protected procedures">
  Require authentication and throw `UNAUTHORIZED` error if session is invalid:

  ```typescript
  export const protectedProcedure = t.procedure.use(({ ctx, next }) => {
    if (!ctx.session || !ctx.session.user) {
      throw new TRPCError({ code: "UNAUTHORIZED" });
    }

    return next({
      ctx: {
        session: { ...ctx.session, user: ctx.session.user },
      },
    });
  });
  ```

  Used for:
  - Creating and editing gigs
  - Sending messages
  - Managing orders
  - Updating profile
</Accordion>

### Client-side usage

Components use the tRPC React Query hooks:

```tsx
'use client';

import { api } from '@/trpc/react';

export function GigList() {
  const { data: gigs } = api.gig.getAll.useQuery();
  
  return (
    // Render gigs
  );
}
```

### Server-side usage

Server Components and Server Actions call procedures directly:

```tsx
import { api } from '@/trpc/server';

export async function GigPage({ params }: { params: { slug: string } }) {
  const gig = await api.gig.getBySlug({ slug: params.slug });
  
  return (
    // Render gig
  );
}
```

## Data flow patterns

### Server Components (default)

Server Components fetch data directly from tRPC:

```tsx
import { api } from '@/trpc/server';

export default async function Page() {
  const data = await api.user.getProfile();
  return <Profile data={data} />;
}
```

### Client Components

Client Components use React Query hooks:

```tsx
'use client';

import { api } from '@/trpc/react';

export function InteractiveComponent() {
  const utils = api.useUtils();
  const { mutate } = api.gig.create.useMutation({
    onSuccess: () => utils.gig.invalidate(),
  });
  
  return <Form onSubmit={(data) => mutate(data)} />;
}
```

### Server Actions

Server Actions handle form submissions and mutations:

```typescript
'use server';

import { db } from '@/server/db';

export async function updateProfile(formData: FormData) {
  // Validate and process
  await db.user.update({
    where: { id: userId },
    data: { name: formData.get('name') },
  });
}
```

## Form handling

Forms use React Hook Form with Zod validation:

<CodeGroup>
```typescript Schema Definition
import { z } from 'zod';

export const LoginSchema = z.object({
  email: z.string().email(),
  password: z.string().min(8),
});
```

```tsx Form Usage
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';

export function LoginForm() {
  const form = useForm({
    resolver: zodResolver(LoginSchema),
  });
  
  return (
    <form onSubmit={form.handleSubmit(onSubmit)}>
      {/* Form fields */}
    </form>
  );
}
```
</CodeGroup>

## Performance optimizations

- **Turbopack**: Faster development builds with Next.js Turbopack
- **Server Components**: Reduce client-side JavaScript
- **React Query caching**: Automatic request deduplication and caching
- **SuperJSON**: Efficient serialization with Date and BigInt support
- **Image optimization**: Next.js automatic image optimization
