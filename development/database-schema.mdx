Khedma Market uses MySQL as its database with Prisma as the ORM. The schema is defined in `prisma/schema.prisma`.

## Database configuration

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}
```

## Core models

### User

The central user model supporting freelancers, clients, companies, and admins.

```prisma
model User {
  id            String    @id @default(cuid())
  name          String?
  firstName     String?   @map("first_name")
  lastName      String?   @map("last_name")
  username      String?   @unique
  email         String?   @unique
  emailVerified DateTime? @map("email_verified")
  image         String?
  password      String?   @db.Text
  role          role?

  // Personal information
  dob        DateTime? @map("date_of_birth")
  bio        String?   @db.Text
  country    String?   @default("Morocco")
  city       String?
  address    String?
  phone      String?
  website    String?
  resume     String?   @db.Text
  profilePic String?   @map("profile_pic")

  // Feature flags
  isTwoFactorEnabled        Boolean @default(false) @map("is_two_factor_enabled")
  inboxNotifications        Boolean @default(true)  @map("inbox_notifications")
  orderUpdatesNotifications Boolean @default(true)  @map("order_updates_notifications")

  // Freelancer profile
  description String?   @db.Text
  
  // Relations
  languages               UserLanguages[]
  education               Education[]
  certifications          Certification[]
  skills                  Skill[]
  sessions                Session[]
  accounts                Account[]
  twoFactorConfirmation   TwoFactorConfirmation?
  conversationsAsSender   Conversation[] @relation("SenderToConversation")
  conversationsAsReceiver Conversation[] @relation("ReceiverToConversation")
  messages                Message[]
  attachements            Attachement[]
  gigs                    Gig[]
  reviews                 Review[]
  orders                  Order[]
  projects                Project[]
  applications            Application[]
  companies               Company[]

  createdAt DateTime @default(now()) @map("created_at")
  updatedAt DateTime @updatedAt @map("updated_at")

  @@map("users")
}
```

<Accordion title="User roles">
  The `role` enum defines four user types:

  ```prisma
  enum role {
    client
    freelancer
    company
    admin
  }
  ```

  - **client**: Can post jobs and order gigs
  - **freelancer**: Can create gigs and apply to jobs
  - **company**: Can post jobs and manage company profile
  - **admin**: Platform administration access
</Accordion>

## Authentication models

NextAuth.js requires several models for authentication:

<CodeGroup>
```prisma Account
model Account {
  id                String  @id @default(cuid())
  userId            String  @map("user_id")
  type              String
  provider          String
  providerAccountId String  @map("provider_account_id")
  refresh_token     String? @db.Text
  access_token      String? @db.Text
  expires_at        Int?
  token_type        String?
  scope             String?
  id_token          String? @db.Text
  session_state     String?
  user              User    @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([provider, providerAccountId])
  @@map("accounts")
}
```

### Order

  completed
  cancelled
}

## Relationships diagram

Key relationships in the schema:

- **User** → One-to-many → Gigs (as owner)
- **User** → One-to-many → Orders (as buyer)
- **User** → Many-to-many → Languages (through UserLanguages)
- **Gig** → One-to-many → Packages
- **Gig** → Many-to-one → Category
- **Order** → Many-to-one → User, Gig, Package
- **Order** → One-to-one → Payment
- **Conversation** → Many-to-one → User (sender and receiver)
- **Message** → Many-to-one → Conversation, User

<Note>
  All relations use `onDelete: Cascade` where appropriate to maintain referential integrity.
</Note>
