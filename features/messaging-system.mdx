## Conversation structure

Each conversation involves two users:

```typescript
model Conversation {
  id         String    @id @default(cuid())
  senderId   String    // User who initiated
  receiverId String    // Other participant
  sender     User      @relation("SenderToConversation", fields: [senderId], references: [id])
  receiver   User      @relation("ReceiverToConversation", fields: [receiverId], references: [id])
  messages   Message[]
  createdAt  DateTime  @default(now())
  updatedAt  DateTime  @updatedAt
}
```

## Starting a conversation

Initiate conversations from multiple places:

**From a gig page**
1. View a freelancer's gig
2. Click "Contact Seller"
3. System creates or opens existing conversation
4. Send your first message

**From a profile**
1. Visit a user's profile
2. Click "Message" button
3. Start chatting immediately

**Creating conversations programmatically**

```typescript
const getOrCreateConversation = async (user1Id: string, user2Id: string) => {
  // Check if conversation exists
  let conversation = await db.conversation.findFirst({
    where: {
      OR: [
        { senderId: user1Id, receiverId: user2Id },
        { senderId: user2Id, receiverId: user1Id }
      ]
    }
  });
  
  // Create new conversation if none exists
  if (!conversation) {
    conversation = await db.conversation.create({
      data: {
        senderId: user1Id,
        receiverId: user2Id
      }
    });
  }
  
  return conversation;
};
```

<Info>
  Only one conversation exists between any two users - previous messages remain accessible even if you haven't chatted in a while.
</Info>

## Sending messages

Messages are the core of each conversation:

```typescript
model Message {
  id             String        @id @default(cuid())
  content        String        @db.Text
  conversationId String
  userId         String        // Message sender
  createdAt      DateTime      @default(now())
  updatedAt      DateTime      @updatedAt
  user           User          @relation(fields: [userId], references: [id])
  conversation   Conversation  @relation(fields: [conversationId], references: [id])
  attachements   Attachement[]
}
```

### Sending text messages

1. Navigate to your inbox
2. Select a conversation
3. Type your message in the input field
4. Press Enter or click Send

```typescript
const sendMessage = async (conversationId: string, content: string, userId: string) => {
  return await db.message.create({
    data: {
      content,
      conversationId,
      userId
    },
    include: {
      user: true
    }
  });
};
```

### Adding attachments

Share files within your messages:

```typescript
model Attachement {
  id        String   @id @default(cuid())
  name      String
  url       String
  type      String   // "document", "image", etc.
  messageId String?
  message   Message? @relation(fields: [messageId], references: [id])
  createdAt DateTime @default(now())
}
```

**To attach files:**

1. Click the attachment icon in the message composer
2. Select files from your computer
3. Files upload automatically
4. Send the message with attachments

**Supported file types:**
- Documents (PDF, DOC, DOCX, TXT)
- Images (JPG, PNG, GIF)
- Archives (ZIP, RAR)
- Other common formats

<Note>
  File attachments are stored securely and remain accessible throughout the conversation history.
</Note>

## Managing conversations

### Viewing your inbox

Access all your conversations from the inbox:

1. Click the messages icon in the navigation
2. See a list of all conversations
3. Conversations show:
   - Other user's name and profile picture
   - Most recent message preview
   - Timestamp of last message
   - Unread indicator (if applicable)

```typescript
const getUserConversations = async (userId: string) => {
  return await db.conversation.findMany({
    where: {
      OR: [
        { senderId: userId },
        { receiverId: userId }
      ]
    },
    include: {
      messages: {
        orderBy: { createdAt: 'asc' },
        select: {
          id: true,
          content: true,
          createdAt: true,
          attachements: true,
          user: {
            select: {
              id: true,
              name: true,
              username: true,
              image: true
            }
          }
        }
      },
      sender: true,
      receiver: true
    }
  });
};
```

### Conversation view

Inside a conversation, you'll see:

**Message thread**
- All messages in chronological order
- Your messages aligned to the right
- Other user's messages aligned to the left
- Timestamps for each message
- Attached files displayed inline

**User information**
- Other user's profile picture
- Username and full name
- Link to their full profile
- Online status (if available)

### Searching conversations

Find specific messages or conversations:

- View conversations with attachments

## Communication guidelines

Follow these guidelines for effective communication:

1. **Keep it professional**: All messages should be work-related and respectful
2. **Don't share sensitive info**: Avoid sharing passwords or financial details in messages
3. **Use attachments**: Share files rather than describing them
4. **Confirm understanding**: Recap important decisions or requirements
5. **Document agreements**: Keep important details in writing

<Note>
  Message history serves as a record of your communication. Reference previous messages to maintain context.
</Note>

## Privacy and security

Your conversations are private:

- Only conversation participants can view messages
- Messages are not visible to other platform users
- Platform administrators have access for support purposes only
- Report inappropriate messages to platform support
